Continuous Integration and Deployment

Continuous Integration and Deployment (aka DevOps) are the principles which are now commonly used in the modern development. When team is working on a complex application and where tasks are spread between developers sometimes it's crucial to be able to test multiple tasks against each other. Therefore teams are in need of a solution to continuously integrate they changes and also being able to deploy it to the enviroments as seamlessly as possible.

This is where CI / CD come in handy.
CI - continuous integration is

...a software development practice where members of a team integrate their work frequently, usually each person integrates at least daily - leading to multiple integrations per day.

Martin Fowler in Continuous Integration

CD - continuous deployment is

...a software development discipline where you build software in such a way that the software can be released to production at any time.

Martin Fowler in Continuous Delivery

In our application we use GitHub to organize CI/CD process via GitHub Actions. This allows us to configurate automatic deployments of our code to our enviroments like DEV and PROD. In GitHub we configure each pull request code to be deployed to the DEV enviroment so we are able to test it before approving the pull request. After the testing and pull request approval our code is getting deployed to the PROD enviroment. This allows us to solely concentrate on the development process rather then managing the code merges.
