Docker

Docker is a helpfull tool in terms of running and deploying your application with pre-packaged configuration.

Here I will provide some highlights on Docker.

Container

Docker container is a unit which contains an app to run as well as app's configuration and enviroment configuration as well. It allows to run the app on the machine by using prepackaged configuration. The main difference from running the apps on separate virtual machines is the fact that you can create separate containers for the separate apps while they all sharing the same virtual machine.

Image

Image is what actually makes up a container. It is a package that contains all of the configuration with the app that then is being used to create a container to run.

Docker commands

Here I would like to list the main commands we were using during the class projects.
  • docker build - allows us to build an imagine from a Dockerfile
  • docker commit - allows us to build an imagine from a container's changes
  • docker run - allows us to run our docker image
  • docker compose - allows us to define and run multi-container application

Start project and access container

To start project you need to execue docker run command. You also need to provide configuration like port mapping, working directory and other. To access container you can follow the address of the machine you are running it on with the port that you've mapped your project to.
