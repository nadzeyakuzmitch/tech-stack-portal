Pylint and other theory

On this page you can learn more about various important terms which are crucial in the design and development process (click in the card to learn more/view the source).

Pylint

Pylint is a source-code, bug and quality checker for the Python programming language. It is named following a common convention in Python of a "py" prefix, and a nod to the C programming lint program. It follows the style recommended by PEP 8, the Python style guide.

Instantiation

Instantiation is the process of creating an actual instance or specific manifestation of an abstraction or template, such as a class of objects or a computer process, in programming. To instantiate is to make such an instance by specifying one specific variation of an object within a class, giving it a name, and putting it in a physical location.

Object

The object can be a collection of variables, functions, and data structures in the object-oriented programming paradigm; in class-based variations of the paradigm, it refers to a specific instance of a class.

Class

A class is a blueprint for constructing objects (a specific data structure), setting starting values for state (member variables or attributes), and implementing behavior in object-oriented programming (member functions or methods).

Namespace

A namespace is a declarative zone that gives the identifiers (names of types, functions, variables, and so on) within it a scope. Namespaces are used to arrange code into logical categories and to avoid name clashes, which might happen when you have many libraries in your code base.

Constructor

In object-oriented programming, a constructor is a particular method of a class or structure that initializes a newly formed object of that kind. The constructor is automatically invoked whenever an object is created. A constructor is similar to an instance method in that it can be used to set the values of an object's members to default or user-defined values. It normally has the same name as the class.

Fixture

For the tests, a fixture provides a defined, dependable, and consistent context. This could include things like the setting or the content. Fixtures describe the stages and data that make up a test's organized phase. They are functions you define in pytest that serve this purpose. They can also be used to specify the active phase of a test.

Type hinting

The term type hinting refers to instructing functions to accept only the specified data type. In a more technical sense, Type Hinting is a technique for forcing a function to accept a specific data type.

Typecasting

Typecasting is the process of converting an object from one data type to another. It is used in computer programming to ensure that a function correctly handles variables. The transformation of an integer into a string is an example of typecasting.

Unit testing

Unit testing ensures that each component of a program functions appropriately in isolation. Before a unit is merged with other units, it should be tested to ensure that it behaves correctly.

Static functions

Static functions are advantageous because they do not require an instantiated member of the class to which they are associated. Static functions can provide functionality for a class without needing the programmer to create an instance of the class first.

Instance method

A method that belongs to instances of a class rather than the class itself is known as an instance method. Simply leave static out of the method heading to define an instance method. You refer to variables and methods in the class by their names, without a dot, in the method specification.

Properties

Properties or features are qualities or features that define a class. While classes are collections of things, an instance is a single object that belongs to a given class.

Static method

A static method is one that belongs to a class rather than to a specific instance of that class. A method can be accessed by any instance of a class, whereas methods defined in an instance can only be accessed by that class's object.

Static properties

Static properties can be accessed without having to create a class instance. The static keyword is used to declare static properties.

Encapsulation

One of the core principles in object-oriented programming is encapsulation (OOP). Let's have a look at how it's done and look at some instances. Encapsulation, like a class in Java, is defined as the idea of bundling data and functions that work on that data into a single unit. This idea is also frequently used to conceal an object's internal representation or state from the outside world. This is known as information concealment.

Inheritance

The mechanism by which one class inherits the characteristics and methods of another class is known as inheritance. The Parent class is the one whose attributes and methods are inherited.

Polymorphism

Polymorphism is a basic notion in object-oriented programming (OOP) that describes instances when something appears in several forms. It is a concept in computer science that you can access objects of various types through the same interface. This interface can be implemented in a variety of ways by each type.

Abstraction

In an OOP language, objects provide an abstraction that hides the implementation details on the inside. You only need to know which methods of the object are available to call and which input parameters are required to start a given function, exactly like the coffee maker in your kitchen.

SOLID

Single Responsibility Principle, Open-Closed Principle, Liskov Substitution Principle (LSP), Interface Segregation Principle (ISP), and Dependency Inversion Principle are the SOLID principles of OOP (DIP). OOP applications are legible, testable, scalable, and manageable thanks to the SOLID principles.

Design pattern

A design pattern is a generic repeatable solution to a typically occurring problem in software design in software engineering. A design pattern isn't a finished design that can be turned into code right away. It's a description or template for solving an issue that may be applied to a variety of scenarios.
